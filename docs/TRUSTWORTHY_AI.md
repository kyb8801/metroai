# Trustworthy AI Policy — MetroAI

> How this project keeps AI outputs verifiable, attributable, and honest.
> 본 문서는 MetroAI가 AI 출력의 신뢰성을 보증하기 위해 실제 코드로 강제하는 정책을 요약한다.

## 1. Data-origin labeling (enforced, not advisory)

Every agent output, connector result, and model evaluation carries an origin label:

| Label | Meaning | May be cited externally? |
|---|---|---|
| `live` | Fetched from a real external source | Yes |
| `stub` | Conservative fallback | No — must be disclosed as stub |
| `synthetic` | Generated by this project | Training/validation only |
| `mixed` | live + synthetic fusion | Both must be disclosed |
| `unknown` | Unverifiable | Never |

Exposed via `AgentResult.payload["is_live"]`, `ConnectorResult.is_live`,
`ModelEvaluation.data_origin`; surfaced in the UI as status dots.

## 2. Honest-metrics rule

- No performance number is published without a reproducible test behind it
  (`tests/`, CI on Python 3.10–3.12).
- Synthetic-data results are always labeled as such — e.g. the audit-risk
  model reports **60.6% ± 3.1pp (5-fold CV, synthetic)**, and an earlier
  unverified sandbox figure was publicly retracted.
  See [`HONESTY_NOTES.md`](HONESTY_NOTES.md) for the full citation rules.

## 3. Tamper-evident outputs

- **Ed25519 digital signatures** (RFC 8032) on computation outputs —
  post-hoc modification is detectable.
- **W3C PROV-O provenance graphs** (JSON-LD): full input → model → output
  lineage, designed so a third-party auditor can replay the chain.

## 4. Uncertainty as a first-class citizen

Point estimates are not answers. The engine reports GUM (ISO/IEC Guide 98-3)
budgets, MCM/QMC verification, and inverse-metrology results ship with
sensitivity-based uncertainty (autodiff Jacobians) and measurement gates.

## 5. Agent transparency & isolation

Agents disclose their data source and fallback state at every call; tool
execution is logged; stub fallbacks are never silently presented as live.

## 6. Known limitations (kept public on purpose)

- Audit-risk model awaits validation on real KOLAS outcomes.
- Some connectors degrade to stubs when upstream pages change.
- Single-maintainer project: security review scope is limited to code and
  dependency hygiene; no formal pentest yet.
