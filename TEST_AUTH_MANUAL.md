```bash
cd /sessions/inspiring-ecstatic-hawking/mnt/Desktop/metroai
```

```bash

```

앱이 열리면 "📐 불확도_계산" 페이지로 이동합니다.

## 테스트 시나리오

### 시나리오 1: 게스트 모드 (로그인 안함)

1. **초기 상태 확인**
   - 사이드바 상단에 로그인/회원가입 탭이 표시됩니다
   - 그 아래에 "👤 게스트 모드" 경고창이 표시됩니다
   - 텍스트: "로그인하면 이용 기록이 저장됩니다."

2. **첫 번째 계산**
   - 템플릿 모드에서 "블록게이지" 선택
   - 기본값으로 "🚀 불확도 계산 실행" 버튼 클릭
   - ✅ 계산이 성공적으로 진행됩니다

3. **두 번째 계산 시도**
   - 다른 템플릿(예: "분동")으로 계산 시도
   - ❌ 다음 오류 메시지가 표시됩니다:
     ```
     게스트는 1번의 계산만 가능합니다. 로그인 후 더 많은 계산을 이용하세요!
     ```

### 시나리오 2: 관리자 계정으로 로그인

1. **로그인**
   - 사이드바 "로그인" 탭 클릭
   - 사용자명: `admin`
   - 비밀번호: `admin123`
   - "로그인" 버튼 클릭

2. **로그인 성공 확인**
   - 사이드바에 다음이 표시됩니다:
     ```
     ✅ Admin 로그인됨
     [🚪 로그아웃 버튼]
     ```
   - 그 아래에:
     ```
     📊 이번 달 남은 계산: 3/3
     ```

3. **계산 3회 실행**
   - 템플릿 모드에서 블록게이지 선택 → 계산 실행 (1/3)
   - 사이드바에서 "📊 이번 달 남은 계산: 2/3" 확인
   - 다른 템플릿으로 계산 실행 (2/3)
   - 사이드바에서 "📊 이번 달 남은 계산: 1/3" 확인
   - 또 다른 템플릿으로 계산 실행 (3/3)
   - 사이드바에서 "📊 이번 달 남은 계산: 0/3" 확인

4. **한도 초과 테스트**
   - 4번째 계산 시도
   - ❌ 다음 오류 메시지:
     ```
     이번 달 계산 한도(3회)를 초과했습니다. 내년 같은 기간에 이용할 수 있습니다.
     ```

5. **로그아웃**
   - "🚪 로그아웃" 버튼 클릭
   - 페이지가 새로고침되고 로그인/회원가입 탭 재표시
   - 사이드바에 다시 "👤 게스트 모드" 표시

### 시나리오 3: 새 계정 회원가입

1. **회원가입 탭 클릭**
   - 사이드바에서 "회원가입" 탭 선택

2. **계정 생성**
   - 사용자명: `testuser123` (입력)
   - 비밀번호: `password123` (입력)
   - 비밀번호 재입력: `password123` (입력)
   - "회원가입" 버튼 클릭

3. **성공 메시지 확인**
   - ✅ "회원가입이 완료되었습니다. 로그인해주세요." 메시지 표시

4. **새 계정으로 로그인**
   - "로그인" 탭으로 이동
   - 사용자명: `testuser123`
   - 비밀번호: `password123`
   - 로그인 성공 확인
   - "✅ testuser123 로그인됨" 표시
   - "📊 이번 달 남은 계산: 3/3" 표시

### 시나리오 4: 여러 모드에서 계산

**직접 입력 모드**
1. 사이드바 "모드 선택"에서 "✏️ 직접 입력" 선택
2. 측정 모델: `a + b + c` 입력
3. 입력량 기호: `a, b, c` 입력
4. 각 입력량에 대해 불확도 정보 입력
5. "🚀 불확도 계산 실행" 클릭 → 사용량 1 증가

**위자드 모드**
1. 사이드바 "모드 선택"에서 "🧙 위자드 모드" 선택
2. 측정 분야 선택
3. 표준 불확도 성분들이 자동으로 제시됨
4. 필요한 항목 선택 및 값 입력
5. "🚀 불확도 계산 실행" 클릭 → 사용량 1 증가

## 버그 테스트

### 테스트 1: 페이지 새로고침 후 로그인 상태 유지
1. 로그인한 상태에서 F5 눌러 새로고침
2. ✅ 로그인 상태가 유지되어야 함 (쿠키에 의해)

### 테스트 2: 다른 페이지에서도 인증 적용
1. 다른 페이지 (예: 2_📊_PT_분석.py)도 수정할 예정
2. 현재는 `1_📐_불확도_계산.py`에만 적용됨

### 테스트 3: 월 리셋 시뮬레이션
1. `usage_data.json`에서 월 값을 이전 달로 변경
   ```json
   "month": "2026-03"
   ```
2. 앱 새로고침
3. ✅ 자동으로 `month: "2026-04"`, `count: 0`으로 리셋되어야 함

## 알려진 제한사항

1. **현재 적용 범위**: `pages/1_📐_불확도_계산.py`만 적용
   - 다른 페이지는 아직 인증 미적용

2. **세션 기반**: Streamlit 세션에 의존
   - 복수 탭/창에서 로그인 상태 공유 안됨

3. **로컬 저장소**: JSON 파일 기반
   - 다중 인스턴스 환경에서 동시성 문제 가능
   - 프로덕션에서는 DB로 변경 권장

## 문제 해결

### 문제: "ModuleNotFoundError: No module named 'streamlit_authenticator'"
**해결**: `pip install -r requirements.txt` 실행

### 문제: 로그인 페이지가 표시되지 않음
**해결**:
1. 페이지 새로고침
2. 브라우저 캐시 삭제
3. 세션 상태 확인

### 문제: 사용량이 저장되지 않음
**해결**:
1. `metroai/auth/` 디렉토리 쓰기 권한 확인
2. `usage_data.json` 파일이 손상되지 않았는지 확인
3. JSON 형식이 유효한지 확인

### 문제: 계산 후 사용량이 안 늘어남
**해결**:
1. 로그인 상태 확인 (username이 None이 아닌지)
2. 에러 메시지가 있는지 확인
3. 앱 로그 확인
